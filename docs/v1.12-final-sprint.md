# SimpleVTube — Final Sprint Plan (v1.12)

Honest accounting of what's getting fixed, what's getting polished, what's
getting genuinely new and ambitious — and what's deliberately staying out
of scope, so nothing here is a silent promise we don't keep.

## 🔴 Real fixes (things we didn't do well)

1. **Orphaned frame files never get cleaned up.** Every time you re-crop or
   remove a frame, the crop editor's exported PNG stays on disk in
   `app-data/frames/` forever — nothing ever deleted the old one. Over a
   long editing session this silently accumulates unbounded disk usage.
   Fixed: removing or replacing a frame now deletes its old file too, but
   ONLY if that file lives inside our own managed `frames/` folder — your
   original source images, wherever they live, are never touched.

## 🟢 Polish (quick, low-risk improvements)

2. **Character & Emote window edge/center snapping.** Dragging either
   window now magnetically snaps to screen edges and center when you get
   close, instead of requiring pixel-perfect manual placement.
3. **Zoom percentage readout in the crop editor** — the slider had no
   numeric label before.
4. **Stream Mode** — one toggle collapses the Control Window down to just
   the essentials (profile, launch/hide, status) for a clean capture-free
   workspace while you're live, without losing access to anything (the
   command palette still reaches every setting).

## 🟠 Ambitious (bigger, genuinely new capability)

5. **Automatic Gain Control (AGC) for the microphone.** A real adaptive
   audio feature: a slow-following peak envelope tracks your typical
   loudness and applies a dynamic gain multiplier so quiet mics and loud
   mics both land in a similar usable range automatically, on top of the
   existing noise gate and decibel scaling. Off by default — it changes
   detection behavior, and existing manually-tuned sensitivity thresholds
   shouldn't silently shift for anyone already happy with their setup.

## ⚪ Explicitly deferred (not in this sprint, and why)

- **Animated GIF/APNG/WebP + sprite-sheet playback, breathing/blink
  animation systems beyond what v1.10/v1.11 already cover** — this is the
  Phase 3 animation engine flagged since the very first roadmap doc. Still
  a genuinely separate subsystem, not a quick add.
- **OBS WebSocket integration, virtual camera/NDI/Spout output** — real
  network protocol work (auth handshakes, scene sync) that I can't verify
  against a live OBS instance from here. Building this blind risks handing
  you something that looks done but silently doesn't work.
- **Stream Deck / MIDI / gamepad input, cloud sync, plugin/scripting
  system, character marketplace** — each is a standalone subsystem on the
  scale of everything we've built so far combined, not a sprint item.
- **Multi-monitor-aware docking, character crop/padding beyond the
  existing crop editor, accessibility/UI-scaling pass** — real, valuable,
  but not free, and not what "final sprint" should be spent on over the
  fixes/polish/AGC above.

If any of the deferred items matter enough to you to be next, say which
one and we scope it properly on its own rather than rushing it in here.
